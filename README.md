# 網頁分頁馬賽克寵物建置計劃書 pixel-tab-pet

## 文件資訊
- 文件名稱：網頁分頁馬賽克寵物建置計劃書
- 版本：v1.0
- 日期：2026-05-13
- 專案型態：Chromium 新分頁擴充套件
- 適用平台：Google Chrome、Microsoft Edge（Chromium）

## 專案概要
本專案目標為開發一套可在 Chrome 與 Edge 共用的自訂新分頁擴充套件，主體為馬賽克像素風互動寵物。技術基礎採用 Manifest V3，透過 `chrome_url_overrides.newtab` 覆寫瀏覽器新分頁，並以擴充套件內部 HTML 作為主入口。

本案的視覺核心明確定義為復古 16-bit 像素風，所有角色、配件與介面裝飾需維持一致的像素網格、色盤密度與邊緣呈現方式。若需在瀏覽器中放大顯示像素素材，應使用 `image-rendering: pixelated` 以保留像素藝術的方塊感並避免平滑模糊。

## 專案目標
### 核心目標
- 建立可於 Chrome 與 Edge 共用的自訂新分頁擴充套件。
- 以馬賽克像素風寵物作為主要互動物件。
- 提供時鐘、快捷入口、背景切換與基本設定。
- 保持新分頁載入快速與低資源消耗，符合 Chromium 擴充套件效能原則。
### 成功標準
- 使用者安裝後可直接以自訂頁面取代預設新分頁。
- 寵物可穩定執行待機、走動、跳躍、睡眠等動畫。
- 像素素材在不同縮放倍率下仍維持清晰邊緣，不出現糊化。
- Chrome 與 Edge 均可正常安裝與執行。

## 視覺與素材規格
### 美術風格定義
本案採用復古 16-bit 馬賽克像素風，畫面需具備以下特徵：
- 清楚像素邊界。
- 禁用反鋸齒與平滑縮放。
- 統一角色比例、色階與亮暗處理。
- 配件與 UI 裝飾遵循相同像素格線。

像素風素材一般適合以 sprite sheet 管理，能有效支援低幀數循環動畫與前端切圖控制，亦常用於瀏覽器 2D 動畫呈現。

### 素材尺寸建議
- 基礎角色尺寸：32x32 或 64x64。
- 配件圖示尺寸：16x16 或 32x32。
- 所有素材以 8 或 16 的倍數切齊。
- 背景與 UI 元件不得破壞角色像素語言一致性。

### 素材內容範圍
目前已規劃之寵物主題素材類型如下：
- 貓：英短、暹羅、緬因。
- 狗：柴犬、柯基、黃金獵犬。
- 鳥：老鷹、貓頭鷹、鸚鵡。
- 魚：金魚、小丑魚、鬥魚。

每組素材建議統一輸出為單張 sprite sheet，內容包含：
- 第一排：待機 4 幀。
- 第二排：走路或游動 4 幀。
- 第三排：跳躍 2 幀、睡覺 2 幀。
- 第四排：配件圖示 4 個。

## 技術架構
### 擴充套件架構
本專案以 Manifest V3 為正式標準，並使用 `chrome_url_overrides.newtab` 設定新分頁入口。Chrome 官方與 MDN 文件均指出，new tab override 需指向擴充套件本地 HTML，而非遠端 URL。

### 前端技術
- HTML：新分頁容器與設定介面。
- CSS：版面、主題、像素字體與縮放控制。
- Vanilla JavaScript：互動行為、狀態切換、設定管理。
- Canvas 2D 或 DOM sprite：寵物動畫層。

若以像素風為主要美術形式，Canvas 與 sprite sheet 是較適合的方式，可清楚控制每幀貼圖與顯示倍率。

### 建議專案結構
```text
pixel-pet-tab/
  manifest.json
  newtab.html
  styles/
    newtab.css
    pet.css
  scripts/
    newtab.js
    pet-engine.js
    state.js
  assets/
    pets/
    icons/
    bg/
```

## 功能規劃
### MVP 功能
- 自訂新分頁首頁。
- 一般資訊區：時間、日期。
- 快捷入口區。
- 背景主題切換。
- 單一寵物載入與互動。
- 設定面板。

### 寵物互動功能
第一版建議狀態如下：
- `idle`：待機動畫。
- `walk` / `swim`：隨機移動或游動。
- `look`：面向滑鼠位置。
- `jump`：點擊觸發。
- `sleep`：閒置後進入睡眠。

建議以有限狀態機（FSM）管理寵物行為，降低不同動畫事件互相衝突的風險。

### 進階功能預留
- 餵食。
- 多品種切換。
- 多寵物輪播。
- 待辦事項或番茄鐘模組。
- 背景主題包與季節造型。

## 效能要求
Chrome 擴充套件效能建議指出，應降低啟動成本、減少不必要運算並延後非必要工作，這對新分頁類產品尤其重要。

本案效能要求如下：
- 新分頁首屏顯示優先。
- 寵物素材預載應控制數量。
- 動畫更新頻率應受控。
- 避免大型框架與過重資源。
- 非必要配件與背景延後載入。

## 相容性與限制
`chrome_url_overrides` 可用於新分頁覆寫，但若系統中存在多個同類型新分頁擴充套件，實際上只會有一個生效。此外，override 內容需為擴充套件內部頁面，無法直接指定遠端網站。

由於 Edge 為 Chromium 架構，整體擴充套件主體可共用，但仍需分別面對不同商店的上架審核與描述設定。

## 打包與匯入流程
### 打包流程
- 專案根目錄必須包含 `manifest.json`，Chrome 官方文件明確要求 manifest.json 位於 extension root directory。
- 每次準備正式版時，需同步更新 `manifest.json` 內的 `version`，避免版本管理混亂。
- 開發與內部測試階段建議以 unpacked extension 為主，便於快速修改與驗證。
- 若需封裝，可透過 Chrome 的打包機制產出 `.crx`；CRX 本質上是已簽署的 ZIP 格式封裝。
- 命令列打包可使用 `chrome.exe --pack-extension=專案路徑 --pack-extension-key=金鑰路徑`，首次打包後會產生 `.crx` 與 `.pem` 私鑰檔；後續更新應保留同一把 `.pem` 才能維持同一 extension 身分。

### Chrome 匯入流程
1. 開啟 `chrome://extensions`。
2. 啟用 **Developer mode**。
3. 點選 **Load unpacked**。
4. 選擇包含 `manifest.json` 的專案資料夾完成載入。

### Edge 匯入流程
1. 開啟 Edge，進入擴充套件管理頁。
2. 啟用 **Developer mode**。
3. 點選 **Load unpacked**。
4. 選擇包含 `manifest.json` 的專案資料夾完成載入。

### 重新載入與更新規則
Chrome 官方文件指出，`manifest`、service worker 與 content scripts 變更後需要重新 reload extension；popup、options page 與其他 extension HTML 頁面則不一定需要整個 extension reload。

### 發佈與分發備註
- 若要正式公開發佈，通常由各自商店上傳封裝內容並處理上架流程，不一定需要自行分發 `.crx`。
- 自行產生 `.crx` 較適合內部測試、灰度版本或非公開發佈情境。

## 風險與對策
### 風險一：像素素材不一致
若不同品種素材採樣風格不一致，將破壞整體產品質感。

對策：
- 建立統一像素格線與色盤規範。
- 每批素材進版前進行視覺檢查。

### 風險二：效能負擔過高
若一次載入多組高解析 sprite，可能造成新分頁初始化延遲。

對策：
- 預設僅載入當前啟用品種。
- 其他素材採延遲載入或切換時讀取。

### 風險三：新分頁擴充套件衝突
若使用者已安裝其他 new tab override 類擴充套件，可能使本產品無法生效。

對策：
- 於安裝說明與提示訊息中明確揭露。
- 提供故障排除文件。

## 定案結論
本案正式定義為「Chrome / Edge 共用之馬賽克像素風互動寵物新分頁擴充套件」，技術採用 Manifest V3 與 `chrome_url_overrides.newtab`，美術採用統一 sprite sheet 像素素材流程。第一版聚焦在自訂新分頁、像素寵物互動、背景與快捷功能，以確保開發可控、效能穩定與後續品種擴充彈性。
